## 2. 核心技术原理

### 2.1 推荐系统的本质

**核心问题**：如何预测用户对物品的喜好程度？

**数学表示**：
```
对于用户u和物品i，预测评分：r̂(u,i) = ?

目标：让预测值r̂尽可能接近真实评分r
```

### 2.2 两种算法的核心思想

#### 2.2.1 ItemCF（协同过滤）

**核心思想**：
> 如果用户喜欢物品A，而物品A和物品B很相似，那么用户很可能也喜欢物品B。

**关键问题**：
1. 如何衡量两个物品的相似度？
2. 如何根据相似物品预测评分？

#### 2.2.2 神经网络

**核心思想**：
> 不依赖人工设计的相似度计算，而是通过数据自动学习复杂的非线性关系。

**关键问题**：
1. 如何将用户和物品表示为向量？
2. 如何学习它们之间的复杂交互？
3. 如何将学习到的表示转换为评分预测？

### 2.3 核心概念对比

| 概念 | ItemCF | 神经网络 |
|-----|--------|---------|
| **特征表示** | 原始评分向量 | 学习的Embedding向量 |
| **相似度计算** | 预定义的公式（余弦/皮尔逊） | 通过网络层学习 |
| **关系建模** | 线性/简单非线性 | 复杂的非线性函数 |
| **参数学习** | 无（或少量） | 大量可学习参数 |
| **可解释性** | 好（可解释相似原因） | 差（黑盒模型） |
| **数据需求** | 较少 | 较多 |
| **计算成本** | 低 | 高 |

## 3. 算法详解

### 3.1 ItemCF算法详解

#### 3.1.1 数据表示

**用户-物品评分矩阵：**

```
        电影A   电影B   电影C   电影D   电影E
用户1    5       3       -       1       -
用户2    4       -       2       -       5
用户3    -       4       5       2       -
用户4    1       -       -       4       3

("-" 表示未评分)
```

#### 3.1.2 相似度计算

**余弦相似度公式：**

```
sim(A, B) = (A · B) / (||A|| × ||B||)

其中：
- A · B = Σ(Ai × Bi)  （点积）
- ||A|| = √(Σ(Ai²))  （向量的模）
```

**计算示例：**

```
电影A = [5, 4, -, 1]  （用户1,2,3,4的评分）
电影B = [3, -, 4, -]

共同评分的用户：只有用户1（A=5, B=3）

sim(A, B) = (5×3) / (√(5²) × √(3²))
          = 15 / (5 × 3)
          = 15 / 15
          = 1.0

注意：由于只有1个共同评分，这个结果不可靠
通常需要至少2-3个共同评分才计算相似度
```

#### 3.1.3 推荐生成

**预测评分公式：**

```
pred(u, i) = Σ(sim(i, j) × r(u, j)) / Σ(sim(i, j))

其中：
- u：目标用户
- i：目标物品（要预测评分的电影）
- j：用户u已评分的物品
- sim(i, j)：物品i和j的相似度
- r(u, j)：用户u对物品j的评分
```

**推荐流程：**

```
1. 找到用户已评分的所有电影
2. 对于每部未评分的电影：
   a. 找到它与用户已评分电影的相似度
   b. 使用加权平均公式预测评分
3. 按预测评分排序
4. 返回Top-N推荐
```

### 3.2 神经网络算法详解

#### 3.2.1 核心思想

**问题：** 传统方法使用预定义的相似度公式，难以捕捉复杂的非线性关系。

**解决方案：** 使用神经网络自动学习：
1. 如何将用户和物品表示为向量（Embedding）
2. 如何学习它们之间的复杂交互（MLP）
3. 如何将学习到的表示转换为评分预测

#### 3.2.2 模型架构

```
输入层                              输出层
  │                                    │
  ▼                                    ▼
┌──────────────┐               ┌──────────────┐
│   用户ID     │               │   预测评分   │
│  (610维)    │               │   (1-5分)   │
└──────┬───────┘               └──────────────┘
       │                              ▲
       ▼                              │
┌──────────────┐                      │
│  用户Embedding│                      │
│   (32维)    │                      │
└──────┬───────┘                      │
       │                              │
       ▼                              │
┌──────────────┐    ┌──────────────┐  │
│    拼接层    │───▶│     MLP      │──┤
│   (64维)    │    │  (64→32→1)   │  │
└──────┬───────┘    └──────────────┘  │
       │                              │
       ▼                              │
┌──────────────┐                      │
│  电影Embedding│                      │
│   (32维)    │                      │
└──────┬───────┘                      │
       │                              │
       ▼                              │
┌──────────────┐               ┌──────────────┐
│   电影ID     │               │   Sigmoid    │
│  (2269维)   │               │   激活函数   │
└──────────────┘               └──────────────┘
```

#### 3.2.3 Embedding层详解

**问题：** 用户ID是离散的（1, 2, 3...），如何表示为连续的向量？

**解决方案：** Embedding层

**原理：**
- 每个ID对应一个可学习的向量
- 向量维度远低于ID数量（32维 vs 610个用户）
- 通过训练自动调整，使得相似的用户/电影有相似的向量

**类比：**
想象Embedding向量是一个人的"基因编码"：
- 每个维度代表一个特征（如喜欢动作片的程度、喜欢喜剧片的程度等）
- 相似的用户会有相似的"基因"
- 模型通过学习这些"基因"来预测用户的喜好

**数学表示：**

```python
# Embedding层
user_embedding = nn.Embedding(num_users=610, embedding_dim=32)
movie_embedding = nn.Embedding(num_movies=2269, embedding_dim=32)

# 前向传播时
user_vector = user_embedding(user_id)    # [batch_size, 32]
movie_vector = movie_embedding(movie_id)  # [batch_size, 32]
```

#### 3.2.4 MLP（多层感知机）详解

**问题：** 如何将用户向量和电影向量转换为评分预测？

**解决方案：** 多层感知机（MLP）

**结构：**

```
输入：拼接向量 [user_vector, movie_vector] (64维)
  ↓
Layer 1: Linear(64 → 64) + ReLU + Dropout
  ↓
Layer 2: Linear(64 → 32) + ReLU + Dropout
  ↓
输出：Linear(32 → 1) + Sigmoid
```

**各层作用：**

1. **Linear（全连接层）**：线性变换 `y = Wx + b`
2. **ReLU（激活函数）**：引入非线性 `f(x) = max(0, x)`
3. **Dropout**：随机丢弃神经元，防止过拟合
4. **Sigmoid**：将输出压缩到[0,1]范围

**为什么需要非线性？**

如果没有激活函数，多层线性变换等价于单层线性变换：
```
y = W2(W1x + b1) + b2 = (W2W1)x + (W2b1 + b2) = W'x + b'
```

激活函数引入非线性，使得网络可以学习复杂的模式。

**代码实现：**

```python
self.mlp = nn.Sequential(
    nn.Linear(64, 64),
    nn.ReLU(),
    nn.Dropout(0.2),
    nn.Linear(64, 32),
    nn.ReLU(),
    nn.Dropout(0.2),
    nn.Linear(32, 1),
    nn.Sigmoid()
)

# 前向传播
concat_vector = torch.cat([user_vector, movie_vector], dim=1)
output = self.mlp(concat_vector)
prediction = output * 4 + 1  # [0,1] → [1,5]
```

#### 3.2.5 训练过程详解

**目标：** 找到最优的参数，使得预测评分尽可能接近真实评分

**损失函数（Loss Function）：**

使用均方误差（MSE）：
```python
loss = (predicted_rating - actual_rating)²
```

**优化器（Optimizer）：**

使用Adam优化器（自适应学习率）：
```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

**训练循环：**

```python
for epoch in range(10):  # 训练10轮
    for batch in dataloader:  # 每次取一个批次
        # 1. 前向传播：计算预测值
        predictions = model(batch.user_ids, batch.movie_ids)
        
        # 2. 计算损失
        loss = MSE(predictions, batch.ratings)
        
        # 3. 反向传播：计算梯度
        loss.backward()
        
        # 4. 更新参数
        optimizer.step()
        optimizer.zero_grad()  # 清零梯度
    
    # 每轮结束后，在测试集上评估模型
    evaluate_on_test_set()
```

**为什么需要多个epoch？**

神经网络需要多次遍历数据才能收敛。每遍历一次（一个epoch），模型就会学到更多模式，参数也会调整得更好。

**批次（Batch）训练：**

为什么不一次性处理所有数据？
- 内存限制：数百万条数据可能无法一次性加载
- 计算效率：GPU并行计算小批次更高效
- 收敛稳定性：小批次的随机性有助于跳出局部最优

## 6. 常见问题解答

### Q1: 为什么神经网络比ItemCF准确率高？

**A:** 神经网络能够：
1. 自动学习用户和物品的潜在特征表示（Embedding）
2. 捕捉复杂的非线性关系（通过多层网络）
3. 泛化到未见过的用户-物品对（更好的泛化能力）

### Q2: 为什么ItemCF的覆盖率那么低？

**A:** ItemCF要求两个物品必须有共同评分者才能计算相似度。在数据稀疏的情况下，很多物品对没有共同评分者，导致无法生成预测。

### Q3: 如何选择使用哪种算法？

**A:** 
- **准确性要求高**：使用神经网络
- **可解释性要求高**：使用ItemCF（可以解释"为什么推荐这部电影"）
- **冷启动场景**：使用神经网络（可以处理新用户/新物品）
- **计算资源有限**：使用ItemCF（训练和推理都更快）

### Q4: 如何改进这个系统？

**A:** 可能的改进方向：
1. **算法层面**：尝试更复杂的神经网络架构（如DeepFM、NCF等）
2. **特征工程**：加入电影类型、演员、导演等元数据
3. **序列推荐**：考虑用户行为的时序特征
4. **部署优化**：使用更高效的推理框架，支持GPU加速
5. **系统功能**：增加用户注册、实时反馈、A/B测试等功能
