## 1. 推荐系统概述

### 1.1 什么是推荐系统？

**推荐系统（Recommendation System）** 是一种信息过滤系统，旨在预测用户对物品的偏好或评分，从而主动向用户推荐其可能感兴趣的物品。

**核心问题：**
- 用户面临海量信息，无法有效筛选
- 平台需要提升用户体验和商业价值
- 建立用户与物品之间的高效连接

### 1.2 推荐系统的核心任务

```
输入：
├── 用户信息（用户ID、历史行为、偏好）
├── 物品信息（物品ID、属性、特征）
└── 交互数据（评分、点击、购买记录）

处理：
└── 推荐算法（协同过滤、神经网络等）

输出：
└── Top-N推荐列表（按预测评分排序）
```

### 1.3 推荐系统分类

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **基于内容** | 分析物品属性和用户偏好匹配 | 无需其他用户数据 | 难以发现新兴趣 |
| **协同过滤** | 发现相似用户或物品的群体行为 | 能发现潜在兴趣 | 数据稀疏性问题 |
| **深度学习** | 自动学习特征表示和复杂模式 | 准确率高、覆盖面广 | 需要大量数据 |

## 2. 协同过滤算法原理

### 2.1 核心思想

**协同过滤（Collaborative Filtering）** 基于一个核心假设：
> 如果两个用户对某些物品的评价相似，那么他们对其他物品的评价也会相似。

**基本流程：**
1. 收集用户历史行为数据（评分、浏览等）
2. 计算用户之间或物品之间的相似度
3. 基于相似度预测用户对未评分物品的评分
4. 生成Top-N推荐列表

### 2.2 基于物品的协同过滤（ItemCF）

#### 2.2.1 算法步骤

```
Step 1: 构建用户-物品评分矩阵
        
        用户1  用户2  用户3  用户4
电影A   [  5     3     -     1   ]
电影B   [  4     -     2     -   ]
电影C   [  -     4     5     2   ]
        
（"-" 表示未评分）

Step 2: 计算物品之间的相似度

对于每对物品，计算它们的相似度：
- 余弦相似度：计算两个物品向量在评分空间中的夹角
- 皮尔逊相关系数：考虑评分的均值差异

相似度公式（余弦相似度）：
sim(i,j) = (向量i · 向量j) / (||向量i|| * ||向量j||)

Step 3: 为目标用户生成推荐

对于用户u和未评分物品i：
1. 找到与用户u已评分物品最相似的K个物品
2. 使用加权平均预测评分：
   
   pred(u,i) = Σ(sim(i,j) * rating(u,j)) / Σ(sim(i,j))
   
   其中j是用户u已评分的物品，sim(i,j)是物品i和j的相似度

Step 4: 排序并返回Top-N推荐
```

#### 2.2.2 相似度计算方法

**余弦相似度（Cosine Similarity）：**

```python
import numpy as np

def cosine_similarity(vec1, vec2):
    """
    计算两个向量的余弦相似度
    值域：[-1, 1]，1表示完全相同，-1表示完全相反
    """
    # 计算点积
    dot_product = np.dot(vec1, vec2)
    
    # 计算向量的模长
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    
    # 避免除以零
    if norm1 == 0 or norm2 == 0:
        return 0.0
    
    # 计算余弦相似度
    similarity = dot_product / (norm1 * norm2)
    
    return similarity
```

**皮尔逊相关系数（Pearson Correlation）：**

```python
def pearson_correlation(vec1, vec2):
    """
    计算两个向量的皮尔逊相关系数
    考虑了评分的均值差异，更适合评分数据
    """
    # 找到共同评分的索引
    mask = (vec1 > 0) & (vec2 > 0)
    
    if np.sum(mask) < 2:
        return 0.0
    
    # 提取共同评分的部分
    common1 = vec1[mask]
    common2 = vec2[mask]
    
    # 计算均值
    mean1 = np.mean(common1)
    mean2 = np.mean(common2)
    
    # 计算协方差和标准差
    covariance = np.sum((common1 - mean1) * (common2 - mean2))
    std1 = np.sqrt(np.sum((common1 - mean1) ** 2))
    std2 = np.sqrt(np.sum((common2 - mean2) ** 2))
    
    if std1 == 0 or std2 == 0:
        return 0.0
    
    correlation = covariance / (std1 * std2)
    
    return correlation
```

#### 2.2.3 优缺点分析

**优点：**
1. **可解释性强：** 可以明确告诉用户推荐某部电影的原因（"因为你喜欢《XXX》，而这两部电影很相似"）
2. **稳定性好：** 物品的相似度相对稳定，不像用户兴趣那样频繁变化
3. **计算效率：** 物品数量通常远少于用户数量，相似度计算更高效

**缺点：**
1. **数据稀疏性：** 用户通常只评价了少量电影，导致相似度计算不准确
2. **冷启动问题：** 对于新电影，缺乏评分数据，无法计算相似度
3. **覆盖率有限：** 只能推荐与用户历史行为相似的物品，难以发现新的兴趣
